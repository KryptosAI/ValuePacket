⚙ How It Works 
  Payer Agent ──openChannel($5)──▶ PaymentChannel ◀──closeChannel── Payee Agent
       │                              contract                           │
       │                                │                                │
       └──POST /price + PaymentProof─────┘────────────────────────────────┘
              (off-chain, EIP-712 signed, 7ms)
A single on-chain openChannel funds thousands of off-chain requests. Each request carries an EIP-712 payment proof — cryptographically signed, instantly verifiable. closeChannel settles it all in one transaction.
♦ Features

Cross-Framework

ElizaOS, G.A.M.E, LangChain, custom agents — plug into any agent runtime with a simple SDK.

⚡

Micropayments

$0.001 per request. 2 on-chain transactions cover thousands of off-chain payments at 7ms latency.

Deployable spending limits, provider allowlists, and rate controls. Your agent, your rules.

Subscriptions

Auto-renewing payment channels with configurable intervals. Set and forget recurring agent payments.
